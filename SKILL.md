# Game Design Document Creater - 游戏策划案生成器

## Overview

通过交互式问答帮助用户生成完整的游戏策划文档(GDD)。支持新建策划案和优化已有策划案两种模式。系统性地收集游戏类型、风格、核心循环、目标用户等信息，最终输出结构化的策划方案。

## When to Use

- 用户输入 `/gamecreater` 命令
- 用户说"帮我设计一个游戏"
- 用户说"生成游戏策划案"
- 用户说"我想做一个XX类型的游戏"
- 用户需要游戏设计文档
- 用户想优化已有的策划案

### 判断规则详解

#### 规则1: 快速模式触发
**条件**: 用户输入包含"按照"/"参考" + 游戏名称
**示例**:
- "按照原神给我设计一个游戏"
- "参考马里奥设计游戏"
- "我想做一个类似塞尔达的游戏"

**行为**: 直接进入快速模式，分析参考游戏

#### 规则2: 无策划案时
**条件**: 项目内没有策划案文件
**行为**: 直接进入新建模式

#### 规则3: 有策划案 + 命令触发
**条件**: 用户输入 `/game-design-document-creater` 命令
**行为**: 询问用户选择新建还是优化

#### 规则4: 有策划案 + 设计类语言
**条件**: 用户说"帮我设计游戏"、"我想设计一个游戏"等
**行为**: 直接进入新建模式

#### 规则5: 有策划案 + 优化类语言
**条件**: 用户说"帮我优化策划案"、"修改一下策划案"等
**行为**: 直接进入优化模式

### 自动分析参考游戏

**立即分析参考游戏并展示：**
```
📊 已分析《参考游戏名》的核心信息：

| 项目 | 分析结果 |
|------|----------|
| **游戏类型** | [类型+细化类型] |
| **美术风格** | [风格描述] |
| **核心循环** | [核心玩法循环] |
| **目标平台** | [平台] |
| **目标用户** | [用户群体] |
| **商业模式** | [模式] |

### 用户选择"修改部分元素"
```

选项:
1. 🎮 游戏类型

8. ➕ 添加更多特色

**仅针对用户选择的方面进行提问，其他保持分析结果**

## Mode A: 新建策划案流程

### Question Flow

按顺序询问以下问题，每个问题提供选项供用户选择：

### 1. 游戏类型（基础问题）
```
问题: 你想制作什么类型的游戏？

选项:
1. 动作游戏 (ACT) - 强调操作和反应
2. 角色扮演 (RPG) - 角色成长和剧情
3. 策略游戏 (SLG/RTS) - 战略规划和资源管理
4. 休闲游戏 - 简单易上手，碎片时间
5. 模拟经营 - 经营管理，资源调配
6. 卡牌/回合制 - 策略卡牌，回合战斗
7. 射击游戏 - 射击战斗
8. 其他类型 - 用户自定义描述
```

### 1.1 射击游戏细化问题
**触发条件：用户选择了"射击游戏"**

```
问题: 射击游戏的视角是？

选项:
1. 第一人称射击 (FPS) - 玩家视角，沉浸感强
2. 第三人称射击 (TPS) - 可见角色，动作感强
3. 俯视角射击 - 类似《孤胆枪手》
4. 横版射击 - 2D横向卷轴
5. 射击+其他玩法混合 - 如射击+RPG、射击+塔防
6. 还没想好，帮我推荐
```

### 1.2 RPG细化问题
**触发条件：用户选择了"角色扮演(RPG)"**

```
问题: RPG的具体类型是？

选项:
1. ARPG (动作RPG) - 即时战斗，操作感强
2. JRPG (日式RPG) - 回合制/半回合制，重剧情
3. MMORPG - 大型多人在线
4. 挂机RPG - 自动战斗，成长为主
5. Roguelike RPG - 随机地牢，永久死亡
6. 还没想好，帮我推荐
```

### 1.3 策略游戏细化问题
**触发条件：用户选择了"策略游戏"**

```
问题: 策略游戏的具体类型是？

选项:
1. 回合制策略 (SLG) - 经典战棋玩法
2. 即时战略 (RTS) - 实时操控，资源竞争
3. 塔防策略 - 防守塔防
4. 模拟经营+策略 - 经营为主，策略为辅
5. 战棋策略 - 回合制战棋
6. 还没想好，帮我推荐
```

### 1.4 动作游戏细化问题
**触发条件：用户选择了"动作游戏"**

```
问题: 动作游戏的具体类型是？

选项:
1. 砍杀类 - 类似《鬼泣》《真三国无双》
2. 平台跳跃 - 类似《超级马里奥》《空洞骑士》
3. 格斗游戏 - 1v1对战
4. 动作冒险 - 开放世界探索+战斗
5. 音乐节奏 - 结合动作的音乐游戏
6. 还没想好，帮我推荐
```

### 2. 游戏风格
```
问题: 游戏的美术风格是怎样的？

选项:
1. 2D像素风 - 复古像素艺术
2. 2D手绘风 - 美术手绘风格
3. 2D卡通风 - Q版卡通风格
4. 3D写实风 - 真实感3D画面
5. 3D低多边形 - Low Poly风格
6. 暗黑风格 - 暗黑哥特风格
7. 国风/水墨 - 中国传统风格
8. 其他风格 - 用户自定义描述
```

### 3. 目标平台
```
问题: 游戏的目标平台是什么？

选项:
1. 微信小游戏 - 轻量级H5游戏
2. 手机APP (iOS/Android) - 移动端原生
3. PC端 - Windows/Mac客户端
4. 网页游戏 - 浏览器运行
5. 主机平台 - PS/Xbox/Switch
6. 多平台 - 跨平台发布
```

### 4. 核心玩法循环
```
问题: 游戏的核心玩法循环是什么？
（玩家在游戏中反复做的事情）

选项:
1. 战斗刷宝 - 打怪→掉落→变强→打更强的怪
2. 关卡挑战 - 挑战关卡→获得奖励→解锁下一关
3. 经营建设 - 收集资源→建设→扩张→更多资源
4. 竞技对抗 - 匹配→对战→获得排名→继续匹配
5. 探索收集 - 探索世界→发现收集→解锁内容
6. 社交互动 - 与玩家互动→建立关系→共同游戏
7. 剧情体验 - 推进剧情→解锁故事→体验结局
8. 其他玩法 - 用户自定义描述
```

**支持多选：提示用户可以用逗号分隔选择多个，如"1,4"**

### 5. 目标用户
```
问题: 游戏的目标用户群体是？

选项:
1. 硬核玩家 - 追求挑战和深度
2. 休闲玩家 - 碎片时间，轻松体验
3. 中度玩家 - 有一定投入，追求成长
4. 全年龄段 - 老少皆宜
5. 特定圈层 - 如二次元、体育迷等
```

### 6. 商业模式
```
问题: 游戏的商业模式是什么？

选项:
1. 免费内购 (F2P) - 免费下载，道具内购
2. 买断制 - 一次付费，完整体验
3. 广告变现 - 免费游玩，广告收入
4. 订阅制 - 按月/年付费
5. 混合模式 - 内购+广告+订阅
```

### 7. 参考游戏
```
问题: 有没有参考的游戏？
（输入游戏名称，或输入"跳过"）
```

### 8. 核心特色
```
问题: 游戏最想突出的特色是什么？
（一句话描述）
```

### 9. 游戏名称
```
问题: 给游戏起个名字吧？
（或输入"跳过"使用默认名称）
```

## Implementation

执行步骤：

1. **检查已有策划案**
   - 使用 Glob 搜索项目内的策划案文件
   - 列出找到的文件供用户选择

2. **快速模式检测**
   - 如果用户说"按照XX游戏设计"或"参考XX游戏"
   - 自动分析参考游戏信息
   - 展示分析结果
   - 询问是否需要修改
   - 根据用户选择生成或针对性提问

3. **模式选择**
   - 有策划案：询问新建还是优化
   - 无策划案：直接新建

4. **新建模式**
   - 欢迎用户
   - 逐个提问（支持细化问题）
   - 记录用户答案
   - 生成文档

5. **优化模式**
   - 列出现有策划案
   - 选择要优化的策划案（支持多选）
   - 选择优化方向（支持多选）
   - 针对性提问
   - 更新文档

6. **后续支持**
   - 询问是否需要深入某个模块
   - 支持迭代修改
