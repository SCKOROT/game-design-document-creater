# Game Design Document Creator

> Claude Code Skill - 游戏策划案生成器

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

## 📖 简介

Game Design Document Creator 是一个 Claude Code Skill，通过智能交互式问答帮助用户生成完整的游戏设计文档 (GDD)。

只需告诉 Claude 你想做什么游戏，它会自动引导你完成整个策划流程。

## ✨ 功能特性

### 🚀 快速模式
说"按照原神给我设计一个游戏"，自动分析参考游戏并生成策划案：
- 自动识别游戏类型、美术风格、核心循环
- 展示分析结果供确认
- 支持选择性修改后生成

### 📝 新建模式
完整的交互式问答流程：
- **11种游戏类型**，每种都有详细细化问题
- **手机游戏屏幕方向询问**（横屏/竖屏/双向支持）
- 使用选项框友好交互
- 支持多选和跳过
- 每个选项附带示例游戏参考

### ✏️ 优化模式
优化已有策划案：
- 自动检测项目内策划案
- 选择性优化特定模块
- 支持批量优化多个策划案

## 🎮 使用方法

### 触发方式

| 方式 | 示例 |
|------|------|
| 命令 | `/gddcreator` |
| 自然语言 | "帮我设计一个游戏" |
| 参考模式 | "按照原神给我设计一个游戏" |
| 优化模式 | "帮我优化策划案" |

### 触发逻辑

```
用户输入 "按照原神设计游戏" ──→ ⚡ 快速模式
                    │
用户输入 "帮我设计游戏" ──→ 📝 新建模式
                    │
用户输入 "优化策划案" ──→ ✏️ 优化模式
                    │
命令 /gddcreator ──→ 有策划案？──→ 询问选择
                              └──→ 无策划案？──→ 📝 新建模式
```

## 📥 安装

将 `SKILL.md` 文件复制到 Claude Code skills 目录：

```bash
mkdir -p ~/.claude/skills/game-design-document-creater
cp SKILL.md ~/.claude/skills/game-design-document-creater/
```

## 📱 手机游戏屏幕方向

当用户选择手机平台时，会自动询问屏幕方向：

| 方向 | 适用游戏类型 | 特点 |
|------|-------------|------|
| **竖屏** | 休闲、卡牌、消除 | 单手操作友好，碎片时间 |
| **横屏** | 动作、射击、RPG | 沉浸感强，操作空间大 |
| **双向支持** | 棋牌、策略 | 灵活切换，适配性广 |

系统会根据游戏类型智能推荐屏幕方向。

## 🎬 使用示例

### 示例1：快速模式

```
用户: 按照原神给我设计一个游戏

助手: 📊 已分析《原神》的核心信息：

| 项目 | 分析结果 |
|------|----------|
| **游戏类型** | 开放世界动作角色扮演 |
| **美术风格** | 3D二次元动漫风 |
| **核心循环** | 探索收集 + 战斗刷宝 + 剧情体验 |
| **目标平台** | 多平台 |
| **屏幕方向** | 横屏（手机端） |
| **商业模式** | 免费内购 + 抽卡系统 |
| **核心特色** | 元素反应战斗 + 开放世界探索 |

请问您是否需要修改这些元素？
1. ✅ 直接生成策划案
2. ✏️ 修改部分元素
```

### 示例2：新建模式（手机游戏）

```
用户: 帮我设计一个手机麻将游戏

助手: 问题 1: 你想制作什么类型的游戏？
用户: 休闲游戏

助手: 问题 1.1: 休闲游戏的具体类型是？
用户: 麻将/棋牌类

助手: 问题 1.2: 麻将游戏的具体玩法是？
用户: 四川血战麻将

助手: 问题 2: 游戏的美术风格是怎样的？
用户: 国风/水墨风

助手: 问题 3: 游戏的目标平台是什么？
用户: 手机APP (iOS/Android)

助手: 问题 3.1: 游戏的屏幕方向是？

- 竖屏 (Portrait) - 单手操作友好，适合休闲游戏
- 横屏 (Landscape) - 沉浸感强，适合动作/射击/RPG游戏
- 双向支持 - 支持横竖屏切换
- 还没想好，帮我推荐

用户: 还没想好，帮我推荐

助手: 根据您的游戏类型（麻将/棋牌），推荐如下：

💡 麻将/棋牌游戏 → 推荐【竖屏】或【双向支持】
- 竖屏：单手操作，适合单局快速对局
- 双向支持：横屏查看更多手牌信息

请问您选择哪种方向？
```

## 📊 问答流程概览

| 序号 | 问题 | 细化问题 |
|:----:|------|----------|
| 1 | 游戏类型 | 射击/RPG/策略/动作/休闲/模拟/卡牌/体育/音乐/恐怖 |
| 2 | 美术风格 | 色调偏向（可选） |
| 3 | 目标平台 | 屏幕方向（手机游戏） |
| 4 | 游戏引擎 | 引擎版本 |
| 5 | 核心玩法循环 | 玩法深度（可选） |
| 6 | 目标用户 | 年龄层（可选） |
| 7 | 商业模式 | 内购类型（可选） |
| 8 | 游戏规模与开发周期 | - |
| 9 | 参考游戏 | - |
| 10 | 核心特色 | - |
| 11 | 游戏名称 | - |
| 12 | 补充信息 | - |
